## 一、版本概述

### 1.1 版本定位

V0.4 是在 MVP（V0.1-V0.3）基础上的首个价值迭代版本。MVP 已验证了核心功能（用户系统、鹦鹉档案、音频上传分类、异常检测、WebSocket 实时推送、用户反馈），V0.4 的重点是 **完善用户体验、提升产品可用性、构建数据价值闭环**。

### 1.2 版本目标

| 目标 | 说明 | 度量指标 |
|------|------|---------|
| 🎯 提升告警触达率 | 异常事件不依赖用户在线才能通知 | 通知到达率 ≥ 95% |
| 🎯 数据可视化 | 让用户直观看到健康趋势 | Dashboard 页面周访问率 ≥ 60% |
| 🎯 降低使用门槛 | 忘记密码可自助找回 | 密码重置成功率 ≥ 90% |
| 🎯 提升分析精度 | 增加异常行为类型覆盖 | 分类准确率 ≥ 85% |

### 1.3 版本范围

| 模块 | 范围 | Story Points |
|------|------|-------------|
| 通知系统 | 邮件通知、站内消息中心 | 13 pts |
| 健康档案 | 历史趋势、评分图表、周期报告 | 13 pts |
| 数据看板 | Dashboard 统计概览 | 8 pts |
| 用户管理 | 密码重置、个人信息编辑 | 8 pts |
| 音频优化 | 新增行为类型、精度提升 | 13 pts |
| **合计** | | **55 pts** |

## 二、用户故事（User Stories）

### 2.1 通知系统模块

#### US-004：邮件通知（P0，5 pts）

**作为饲养者，我希望在检测到鹦鹉异常行为时收到邮件通知，以便即使不在 App 中也能及时知晓。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-004-1 异常事件触发 | 系统检测到夜间尖叫 | 30 秒内发送邮件到用户绑定邮箱 | 邮件包含：鹦鹉名称、事件类型、时间、风险等级 |
| AC-004-2 连续异常告警 | 1 小时内检测到 3 次以上异常 | 立即发送紧急告警邮件 | 邮件标题标记"⚠️ 紧急"，正文建议联系兽医 |
| AC-004-3 通知频率限制 | 同一鹦鹉 10 分钟内多次异常 | 合并为一条通知 | 避免邮件轰炸 |
| AC-004-4 通知设置 | 用户进入通知设置页 | 开关"邮件通知"选项 | 关闭后不再发送邮件 |
| AC-004-5 邮件模板 | 发送邮件 | 使用品牌化 HTML 模板 | 包含 ParrotCare Logo、事件摘要、跳转链接 |

#### 依赖
- 需要 SMTP 服务配置（开发环境可使用 Mock）
- 用户邮箱字段需标记为"已验证"状态

#### US-005：站内消息中心（P0，5 pts）

**作为饲养者，我希望有一个消息中心集中查看所有通知和系统消息，以便不会遗漏重要信息。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-005-1 消息列表 | 用户进入消息中心 | 显示所有未读消息（最新在前） | 每条消息显示：类型图标、标题、摘要、时间 |
| AC-005-2 未读标记 | 有未读消息 | 导航栏"消息"图标显示红点 + 数量 | 点击后清除未读标记 |
| AC-005-3 消息分类 | 消息中心有不同类型消息 | 可按"异常告警""系统通知""健康提醒"筛选 | 筛选后列表更新 |
| AC-005-4 一键已读 | 用户有多条未读消息 | 点击"全部标记为已读" | 所有未读消息变为已读 |
| AC-005-5 消息详情 | 用户点击一条异常告警消息 | 跳转到对应事件的详情页 | 关联事件正确展示 |
| AC-005-6 消息保留 | 消息产生 | 保留 30 天 | 超过 30 天自动归档 |

#### 依赖
- 需要在数据库新增 `notifications` 表
- 与 WebSocket 推送集成（在线用户即时推送，离线用户存入消息中心）

#### US-006：Web Push 浏览器通知（P1，3 pts）

**作为饲养者，我希望浏览器能推送通知（即使页面在后台），以便在电脑前工作时也能及时收到告警。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-006-1 授权请求 | 用户首次访问 | 弹出浏览器通知权限请求 | 用户同意后启用 |
| AC-006-2 推送通知 | 系统检测到异常且用户已授权 | 浏览器弹出系统级通知 | 显示：鹦鹉名称 + 事件类型 |
| AC-006-3 点击跳转 | 用户点击通知 | 打开对应事件详情页面 | 自动聚焦到该事件 |
| AC-006-4 静默模式 | 用户开启免打扰时段 | 在设定时间内不推送 | 消息仍存入消息中心 |

### 2.4 用户管理模块

#### US-011：密码重置（P0，5 pts）

**作为饲养者，我希望忘记密码时能通过邮箱重置密码，以便恢复账号访问。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-011-1 发起重置 | 用户在登录页点击"忘记密码" | 输入注册邮箱并提交 | 发送密码重置邮件 |
| AC-011-2 重置链接 | 用户收到重置邮件 | 点击邮件中的链接 | 打开重置密码页面（链接有效期 1 小时） |
| AC-011-3 设置新密码 | 用户在重置页输入新密码 | 提交（需确认密码一致） | 密码更新成功，自动跳转登录页 |
| AC-011-4 密码强度 | 用户设置新密码 | 密码长度 < 8 或不含字母+数字 | 提示"密码至少 8 位，需包含字母和数字" |
| AC-011-5 安全限制 | 用户已使用过该重置链接 | 再次访问同一链接 | 提示"链接已失效，请重新发起重置" |
| AC-011-6 频率限制 | 1 小时内发起 3 次重置 | 第 4 次请求 | 提示"请求过于频繁，请稍后再试" |

#### 依赖
- 需要 SMTP 服务发送邮件
- 需要在数据库存储密码重置 Token（带过期时间）

### 2.5 音频分析优化模块

#### US-013：新增行为类型识别（P2，8 pts）

**作为饲养者，我希望系统能识别更多鹦鹉行为类型，以便更精确地了解鹦鹉状态。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-013-1 梳理羽毛 | 上传梳理羽毛音频 | 系统识别为"preening" | 置信度 ≥ 0.75，不标记为异常 |
| AC-013-2 进食声音 | 上传进食音频 | 系统识别为"eating" | 置信度 ≥ 0.75，不标记为异常 |
| AC-013-3 求偶鸣叫 | 上传求偶音频 | 系统识别为"courtship_call" | 置信度 ≥ 0.7，不标记为异常 |
| AC-013-4 恐惧尖叫升级 | 上传持续性高频尖叫 | 系统识别为"distress_scream" | 与一般尖叫区分，标记为 high 风险 |
| AC-013-5 静默检测 | 连续 30 分钟无声音 | 系统记录为"silence_period" | 超过 2 小时标记为潜在异常 |

#### US-014：分类精度优化（P2，5 pts）

**作为系统维护者，我希望利用用户反馈数据持续优化分类模型，减少误判和漏判。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-014-1 反馈统计 | 系统积累 100+ 条用户反馈 | 生成分类精度报告 | 各类型准确率、召回率 |
| AC-014-2 规则调优 | 某类型误判率 > 20% | 系统提示管理员调整规则阈值 | 提供建议阈值范围 |
| AC-014-3 置信度校准 | 历史数据显示某区间置信度与实际不符 | 调整置信度计算逻辑 | 置信度与实际准确率相关性提升 |

## 六、API 变更

### 6.1 新增 API

#### 通知模块 `/api/notifications`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | `/` | 获取消息列表（支持分页、筛选） | JWT |
| GET | `/unread-count` | 获取未读消息数 | JWT |
| POST | `/{id}/read` | 标记单条消息为已读 | JWT |
| POST | `/read-all` | 全部标记为已读 | JWT |

#### 密码重置 `/api/auth/password-reset`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/request` | 发起密码重置请求 | 无（公开） |
| POST | `/confirm` | 确认重置（token + 新密码） | 无（公开） |

#### 健康趋势 `/api/parrots/{parrot_id}/health`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | `/trend` | 获取健康评分趋势数据 | JWT |
| GET | `/report` | 获取周期行为统计报告 | JWT |
| GET | `/heatmap` | 获取 24 小时行为热力图数据 | JWT |

#### Dashboard `/api/dashboard`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | `/` | 获取 Dashboard 全部指标 | JWT |

### 6.2 修改 API

## 八、边界情况 & 风险

| 编号 | 风险/边界 | 影响 | 建议 |
|------|----------|------|------|
| R-001 | SMTP 服务配置 | 邮件通知依赖外部 SMTP | 开发环境使用 Mock，生产环境配置企业 SMTP 或第三方服务（如 SendGrid） |
| R-002 | 浏览器通知兼容性 | Safari/iOS 对 Web Push 支持有限 | 作为增强功能，不替代邮件通知 |
| R-003 | 图表库体积 | 引入 Chart.js 增加前端加载 | 使用 CDN + 按需加载，控制增量 < 100KB |
| R-004 | 数据量增长 | 通知表 30 天积累大量数据 | 定期归档过期数据，考虑分表策略 |
| R-005 | 密码重置邮件被拦截 | 邮件进入垃圾箱 | 使用可信 SMTP，配置 SPF/DKIM 记录 |
| R-006 | 健康评分算法争议 | 用户对扣分值有异议 | V0.4 保持现有算法，V1.0 引入可自定义权重 |
