# ParrotCare 架构说明

## 目标

ParrotCare 使用一套 React PWA 同时服务桌面端和移动端。移动端不是后端 Web 页的缩小版，而是能够独立运行的本地优先应用：没有网络、FastAPI、PostgreSQL 或 MinIO 时，核心护理记录流程仍然成立。

## 当前数据流

```text
页面 (React)
  ↓ 领域操作
localRepository
  ├─ events store: 事件、标签、状态、备注
  └─ audio store: MediaRecorder / 导入文件的 Blob
          ↓
       IndexedDB
```

页面不关心数据来自 IndexedDB 还是未来的远端同步。所有持久化操作经过 `apps/web/src/data/localRepository.ts`，这样可以测试事务边界，也可以在不重写 UI 的情况下加入同步 adapter。

## 平台策略

### 移动端

- PWA manifest 与 Service Worker 缓存应用壳。
- Hash 路由支持静态文件托管、子目录部署和离线启动。
- MediaRecorder 负责录音；不支持录音的浏览器仍可导入音频。
- 事件和音频只写入当前浏览器的 IndexedDB。
- 页面底部固定主导航，交互目标按触摸场景设计。

安装 PWA 与使用麦克风需要安全上下文：生产环境使用 HTTPS；本机开发可使用 localhost。浏览器清除站点数据会删除本地记录，因此后续应增加显式导出/备份功能。

### 桌面端

- 使用相同领域模型和 repository。
- 宽屏布局展示摘要、筛选、记录列表和详情侧栏。
- FastAPI 可以继续提供账号、多设备同步、模型分析与通知，但这些属于增强能力。

## 可选同步边界

若后续接入后台，新增 `RemoteSyncAdapter`，由应用服务协调：

1. 先完成 IndexedDB 本地事务并返回成功。
2. 将待同步操作加入本地队列。
3. 在线时异步推送到后端。
4. 冲突按 `updatedAt` 或版本号处理并保留人工选择入口。
5. 登录失效、超时或服务不可用不得破坏本地数据。

禁止让页面同时调用本地 repository 和远端 API，否则会产生双写、错误状态与无法离线测试的问题。

## 数据与隐私

- 音频 Blob 保存在 IndexedDB，不写入 localStorage。
- 当前版本不会自动上传录音。
- 示例事件不含真实音频。
- 护理提示仅用于记录辅助，不构成医疗诊断。

## 验证矩阵

| 场景 | 必验结果 |
| --- | --- |
| 首次打开 | 自动生成示例记录，页面无后端请求 |
| 录音 | 允许权限后可停止、回放并保存 |
| 导入 | 音频可保存、回放并重新标注 |
| 持久化 | 刷新或关闭重开后记录、状态和备注仍存在 |
| 离线 | 已加载过应用后，应用壳和本地记录可用 |
| 响应式 | 360px 无横向滚动；桌面布局不被移动样式限制 |
| 无麦克风 | 显示可操作错误，并保留导入路径 |
