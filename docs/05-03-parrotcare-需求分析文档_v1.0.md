## 一、项目概述

### 1.1 产品现状

宠物鹦鹉饲养者难以全天候监测鹦鹉行为，尤其夜间异常行为（尖叫、扑翅、撞笼）往往在主人发现时已造成严重后果（羽毛脱落、受伤、心理压力）。市场上缺乏针对鹦鹉行为的专业化 AI 监测解决方案。

### 1.2 业务目标

- **核心目标**：通过 AI 音频分析技术，实时识别鹦鹉异常行为，及时推送告警通知
- **次要目标**：积累行为数据，形成健康档案和趋势分析
- **长期目标**：构建鹦鹉行为知识图谱，提供科学饲养建议

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 个人饲养者 | 1-3 只宠物鹦鹉，手机用户 | 便捷记录、异常通知、行为统计 |
| 专业繁殖者 | 5+ 只鹦鹉，需要多设备管理 | 批量管理、历史数据、设备联动 |
| 兽医诊所 | 需要观察患者行为模式 | 数据导出、趋势报告 |

## 二、用户需求

### US-101：用户注册与登录（Must，3 pts）

**作为饲养者，我希望能够注册账号并安全登录，以便管理我的鹦鹉档案。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-101-1 手机号注册 | 用户打开注册页面 | 输入手机号+密码并提交 | 注册成功并自动登录 |
| AC-101-2 手机号+密码登录 | 已注册用户 | 输入正确的手机号和密码 | 登录成功，获取 Token |
| AC-101-3 错误密码 | 用户输入错误密码 | 提交登录 | 提示"手机号或密码错误" |
| AC-101-4 手机号唯一性 | 使用已注册手机号 | 尝试再次注册 | 提示"该手机号已注册" |

### US-102：鹦鹉档案管理（Must，5 pts）

**作为饲养者，我希望为每只鹦鹉创建健康档案，以便跟踪个体行为。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-102-1 创建档案 | 用户进入鹦鹉管理页 | 填写名称、品种、年龄、性别等并提交 | 创建成功，返回鹦鹉详情 |
| AC-102-2 品种必填 | 创建档案时未填写品种 | 提交 | 提示"品种为必填项" |
| AC-102-3 查看档案列表 | 用户有多只鹦鹉 | 进入管理页 | 显示所有鹦鹉卡片 |
| AC-102-4 健康指标 | 查看鹦鹉详情 | 进入档案详情 | 显示健康评分、今日鸣叫次数、尖叫次数、异常事件数 |

### US-103：音频上传与分类（Must，8 pts）

**作为饲养者，我希望上传鹦鹉的音频记录，系统能自动分类行为类型。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-103-1 上传音频 | 用户选择音频文件 | 上传并指定鹦鹉和时间 | 系统自动分类行为类型 |
| AC-103-2 行为分类 | 上传正常鸣叫音频 | 系统处理 | 分类为"normal_chirp"，置信度 > 0.8 |
| AC-103-3 尖叫检测 | 上传尖叫音频 | 系统处理 | 分类为"scream"，标记为异常事件 |
| AC-103-4 扑翅检测 | 上传扑翅音频 | 系统处理 | 分类为"wing_flap" |
| AC-103-5 撞笼检测 | 上传撞笼音频 | 系统处理 | 分类为"cage_bump"，标记为异常事件 |

### US-104：夜间异常检测（Must，5 pts）

**作为饲养者，我希望系统在夜间检测到鹦鹉异常行为时自动通知。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-104-1 夜间定义 | 时间 22:00-06:00 | 检测到异常音频事件 | 标记为夜间异常事件 |
| AC-104-2 异常等级 | 检测到尖叫 | 风险等级设为"medium" | 显示橙色警告 |
| AC-104-3 连续异常 | 夜间检测到 3 次以上异常 | 风险等级设为"high" | 显示红色告警 + 推送通知 |
| AC-104-4 健康评分扣减 | 发生异常事件 | 更新当日健康评分 | 根据事件严重程度扣减 |

### US-105：事件列表查看（Must，3 pts）

**作为饲养者，我希望查看所有鹦鹉的行为事件记录，以便回顾分析。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-105-1 事件列表 | 用户进入事件页 | 显示最近 7 天的事件 | 按时间倒序排列 |
| AC-105-2 筛选 | 用户选择特定鹦鹉 | 只显示该鹦鹉的事件 | 列表更新 |
| AC-105-3 异常标记 | 事件为异常类型 | 事件卡片显示异常标识 | 红色/橙色标记 |
| AC-105-4 事件详情 | 用户点击事件 | 显示详情（时间、类型、置信度、音频回放） | 详情弹窗 |

### US-106：用户反馈（Should，3 pts）

**作为饲养者，我希望对 AI 分类结果进行反馈，帮助系统学习。**

#### 验收标准

| AC | Given | When | Then |
|----|-------|------|------|
| AC-106-1 反馈按钮 | 用户查看事件详情 | 点击"反馈"按钮 | 弹出反馈表单 |
| AC-106-2 纠正标签 | 用户认为分类错误 | 选择正确行为类型并提交 | 记录反馈数据 |
| AC-106-3 补充说明 | 用户填写评论 | 提交反馈 | 评论保存到反馈记录 |

## 三、非功能需求

| 编号 | 需求 | 标准 |
|------|------|------|
| NF-001 | 性能 | API 响应时间 < 500ms（P95） |
| NF-002 | 音频处理 | 上传到分类完成 < 3 秒（10 秒以内音频） |
| NF-003 | 可用性 | 系统可用性 ≥ 99.5% |
| NF-004 | 安全性 | 用户密码加密存储（bcrypt），JWT Token 有效期 24h |
| NF-005 | 数据存储 | SQLite（MVP 阶段），支持后续迁移到 PostgreSQL |

## 四、数据模型

### 4.1 实体关系

```
User (1) ──< Parrot (1) ──< MediaEvent
  │                            │
  └──< Device                  └──< UserFeedback
  │
  └──> BehaviorDailyStat (via Parrot)
```

### 4.2 核心字段

| 实体 | 关键字段 | 说明 |
|------|---------|------|
| User | user_id, phone, email, subscription_status | 用户基本信息 |
| Parrot | parrot_id, name, species, age, gender, weight, health_history | 鹦鹉档案 |
| Device | device_id, device_type, status, last_online_time | 监测设备 |
| MediaEvent | event_id, event_type, is_abnormal, risk_level, confidence | 行为事件 |
| UserFeedback | feedback_id, feedback_type, feedback_label, comment | 用户反馈 |
| BehaviorDailyStat | stat_date, chirp_count, scream_count, health_score | 每日统计 |
